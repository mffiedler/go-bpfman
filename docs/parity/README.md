# Rust parity evidence

This directory holds reproducible evidence comparing go-bpfman against
Rust bpfman across the shared CLI surface (`load`, `attach`, `detach`,
`unload`, `list`, `get`). It exists so the parity claim can be audited
from real command output rather than taken on trust.

Everything here was generated by hand-driving both binaries; the driver
scripts are committed so you can regenerate it.

## What is here

| Path | What it is |
| --- | --- |
| `RESULTS.md` | Per-scenario pass/diverge tables (the summary to read first). |
| `ISSUES.md` | The behavioural differences found, with reproductions. Starts with a calibration note on what "divergence" does and does not claim. |
| `ENVIRONMENT.md` | Host, binaries + revisions, state isolation, and the veth / netns setup recipes. |
| `outputs/` | Raw transcripts (`<case>.go.out` / `<case>.rust.out`): every command and its output and exit code. |
| `help/` | Captured `--help` for every relevant subcommand of both CLIs. |
| `harness.sh` | Core helpers: `gobpf`/`rsbpf` wrappers, `cap`/`capq` transcript capture, `veth_up`/`veth_down`, JSON/text field extractors, lifecycle `*_begin`/`*_finish`. |
| `baselines.sh` | `baseline TYPE` / `baselines_all` -- full lifecycle per program type. |
| `ordering.sh` | `chain_disp` / `chain_tcx` -- priority ordering chains. |
| `slotreuse.sh` | `slot_reuse` / `slot_exhaust` -- dispatcher slot management. |
| `extra.sh` | tie-break, detach-from-middle, multi-interface, `--netns`. |
| `scenarios.sh` | error / idempotency / `--global` / `--application` / `--map-owner-id` / section-format / probe-flag / image-load captures. |
| `run-all.sh` | Sources all of the above and regenerates every transcript. |

## Setup

Both binaries are referenced through stable symlinks in `bin/` (which
is gitignored):

```sh
# from the go-bpfman repo root
make STAMP=1 bpfman-build                              # builds bin/bpfman (Go)
ln -sfn ./bpfman bin/go-bpfman                          # Go alias
ln -sfn /path/to/rust/target/debug/bpfman bin/bpfman-rs # Rust build
```

The Rust binary used here is the worktree build at
`~/src/github.com/bpfman/bpfman/worktrees/general/target/debug/bpfman`
(`bpfman 0.6.0`). Go is the stamped `make STAMP=1 bpfman-build` output.
Exact revisions are recorded in `ENVIRONMENT.md` and at the top of
`RESULTS.md`.

## Reproducing

Prerequisites: passwordless `sudo`, `jq`, `ip`, `bpftool`, and the
compiled testdata objects under `e2e/testdata/bpf/` (built by the
repo's normal `make` flow). Then:

```sh
bash docs/parity/run-all.sh
```

That regenerates everything under `outputs/`. To run one slice, source
the harness and a driver and call its function, for example:

```sh
source docs/parity/harness.sh
source docs/parity/baselines.sh
baseline kprobe e2e/testdata/bpf/kprobe_counter.bpf.o kprobe:kprobe_counter
```

State isolation (so the runs do not disturb a running bpfman or each
other): Go uses `--runtime-dir /run/bpfman-parity-go`; Rust uses its
default `/run/bpfman`. Every scenario cleans up after itself (detach,
unload, delete interfaces/namespaces). See `ENVIRONMENT.md`.

## Reading a result

`PARITY` means the observable semantic state matched; `DIVERGE` means it
did not. `DIVERGE` is an observation, not a verdict on which side is
correct -- read the calibration note at the top of `ISSUES.md` before
drawing conclusions. Representation differences that are semantically
equal (Go JSON vs Rust text, numeric vs named `proceed_on`, etc.) are
listed separately in `RESULTS.md` and are not counted as divergences.
