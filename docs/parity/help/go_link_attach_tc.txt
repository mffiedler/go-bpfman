$ go-bpfman link attach tc --help
Usage: bpfman link attach tc --priority=INT <program-id> <iface> <direction> [flags]

Attach a TC program to a network interface.

Arguments:
  <program-id>    Program ID to attach.
  <iface>         Network interface.
  <direction>     Direction (ingress or egress).

Flags:
  -h, --help                     Show help (-h for compact, --help for full).

  -o, --output=text              Output format: text, json.
  -m, --metadata=METADATA,...    KEY=VALUE link metadata (can be repeated).
  -p, --priority=INT             Priority in chain (lower runs first;
                                 non-negative). Slot exhaustion (more than 10
                                 attachments) is reported by the dispatcher,
                                 not by this flag.
      --proceed-on=pipe,dispatcher_return,...
                                 TC actions to proceed on (comma-separated or
                                 repeated). Values: unspec, ok, reclassify,
                                 shot, pipe, stolen, queued, repeat, redirect,
                                 trap, dispatcher_return.
  -n, --netns=STRING             Network namespace path.

Global Flags:
  --config=FILE              Config file path (default: /etc/bpfman/bpfman.toml)
                             ($BPFMAN_CONFIG).
  --log=SPEC                 Log spec (e.g., 'info,manager=debug')
                             ($BPFMAN_LOG).
  --runtime-dir=DIR          Root directory for runtime files.
  --image-cache-dir=DIR      Root directory for OCI image cache.
  --lock-timeout=DURATION    Timeout for acquiring the global writer lock (0 for
                             indefinite) ($BPFMAN_LOCK_TIMEOUT).
